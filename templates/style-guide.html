Design system

Style guide

Stanza is a minimal personal publishing system: one voice, large confident typography, a calm canvas, and a warm orange action color.

Design principles

Use these rules when extending the theme. They are the core constraints that keep Stanza recognizable after the original design files are gone.

01

One author first

The layout is for a personal site, not a magazine. One large identity moment, one article title, one subscribe path.

02

Type carries the brand

Display text uses weight 800, tight line-height, and fluid sizes. Avoid decorative effects when type can do the work.

03

Orange is the yes moment

Use brand orange for primary CTAs, content-link underlines, selected states, and featured category chips. Do not use it as a page background.

04

Calm before variety

Cards stay sharp and mostly borderless. Supporting colors appear in category chips, author accents, and small editorial references.

Bundled fonts

The theme ships three font families. Body and headings use the active typography mode through --font-active; code always uses the mono family.

Production note

The original design exploration used slight negative tracking on some display text. The WordPress theme keeps letter spacing at 0 so long titles, translated strings, and Site Editor text wrap predictably.

Category colors

Categories use a curated six-color palette. The colors share a similar visual weight, while orange remains the loudest and should be reserved for featured or primary labels.

Featured

--tag-brand for featured or new posts.

Members

--tag-clay for subscribers and membership.

Reading

--tag-mustard for books and reading lists.

Projects

--tag-sage for projects and evergreen content.

Photography

--tag-petrol for travel, photo, and media-led posts.

Notes

--tag-plum for notes and personal updates.

Adding another category color

Pick a hue not already used, avoid pure red and neon green, keep saturation around 30-65%, keep lightness around 35-45, and compare it against the palette before adding a new --tag-* token. If a site needs more than 8-10 colors, group categories instead of creating a rainbow.

Core components

These are the recurring UI surfaces. Keep them simple, direct, and consistent across templates.

Buttons

Primary buttons use orange, dark text, 40px radius, and 40-48px height. Secondary buttons use transparent or quiet surface fills.

Subscribe input

The most important home-page control. Use the visual wrapper with a newsletter or membership plugin that owns form submission.

Header

Logo, navigation, search, color-mode toggle, sign-in, and subscribe. The background-hero page uses the overlay header; other pages use the regular header.

Article meta

Author image, name, date, category chip, and tags sit in the meta area. Avatar fallback defaults to brand orange.

Layout rules

Container

Max width is 1440px with a fluid outer gap. Avoid narrow decorative cards around whole page sections.

Articles

Post pages use a 12-column canvas. The meta sidebar sits in columns 1-3; content sits in columns 4-9; wide media can span farther.

Responsive

Key changes happen below 992, 840, 768, and 600px. Classic feed cards stack below tablet widths.

Imagery

Use one strong full-bleed photo when imagery is present. Avoid decorative textures, patterns, and illustration backgrounds.

Voice and copy

Stanza copy is direct, first-person, and sentence case. It should sound like a person speaking to a reader, not a platform marketing page. This page is also bundled as the Style Guide page template, so it can be recreated from the Site Editor on a fresh install.

Do

  • Use sentence case: Subscribe, Sign in, Style guide.
  • Write in first person where it fits.
  • Use Unicode arrows for movement: ← and →.
  • Keep excerpts to one clear sentence.

Do not

  • Use title case like a magazine system.
  • Replace subscribe with hype copy.
  • Use emoji as UI icons or list markers.
  • Add decorative flourishes when spacing is enough.
